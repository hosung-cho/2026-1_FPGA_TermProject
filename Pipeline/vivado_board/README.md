# RV32I LeNet Board Build

Target FPGA part: `xc7z020clg484-1`.

This directory is for a real FPGA board build. The part number is enough for
Vivado device selection, but it is not enough for a safe bitstream. You still
need the board's real pin constraints.

## Files

- `run_board_impl.tcl`: Vivado batch build for `RV32I_Board_Top`.
- `run_board_synth_check.tcl`: synthesis-only check that does not require
  board pins. Use this before `board.xdc` is ready.
- `run_axi_synth_check.tcl`: synthesis-only check for `RV32I_AxiLite_Top`,
  the PS/PL internal-connection wrapper.
- `run_ps_bd_check.tcl`: creates a Zynq PS block design, connects
  `RV32I_AxiLite_Top` through AXI GP0, synthesizes the wrapper, and exports a
  Vivado 2019.1 SDK handoff `.hdf`.
- `run_ps_bd_impl.tcl`: builds the same PS/PL block design through
  implementation and writes a bitstream.
- `run_ps_loadable_bd_impl.tcl`: builds the PS/PL sample-load variant where
  the PS can replace the 32x32 input image and expected label before each run.
- `preflight_board.ps1`: checks that the RTL, firmware images, and board build
  scripts are present.
- `board_template.xdc`: pin constraint template. Copy it to `board.xdc`.
- Generated by the Tcl script:
  - `imem.hex`: copied from `FPGA_proj/firmware/lenet_infer_imem.hex`
  - `lenet_digit7_dmem.mem`: copied from `FPGA_proj/firmware/lenet_digit7_dmem.mem`
  - `build/rv32i_lenet_board.bit`

## LED Mapping

- `LED[0]`: LeNet firmware wrote done flag.
- `LED[4:1]`: predicted digit.
- `LED[5]`: pass, predicted digit is 7 for the bundled digit7 input.
- `LED[6]`: fail, predicted digit is not 7.
- `LED[7]`: heartbeat.

## PS/PL AXI-Lite Wrapper

`Pipeline/src/rtl/RV32I_AxiLite_Top.v` is the no-external-PL-pin path. Put it
inside a Zynq block design and connect:

- `s_axi_aclk` to PS `FCLK_CLK0`
- `s_axi_aresetn` to the corresponding active-low interconnect reset
- `s_axi_*` to a PS AXI master through AXI interconnect/smartconnect

Readable registers:

```text
0x00 control: bit0 run_enable, bit1 write-1 clear status
0x04 status : bit0 done, bit1 pass, bit2 fail
0x08 predicted digit
0x0C expected digit
0x10 heartbeat
0x14 debug_pc
0x18 debug_inst
0x1C debug_data_addr
0x20 debug_read_data
0x24 debug_write_data
0x28 debug_data_we
```

## Run

1. Run the file preflight:

```powershell
cd Pipeline\vivado_board
powershell -NoProfile -ExecutionPolicy Bypass -File .\preflight_board.ps1
```

2. Optionally run synthesis without board pins:

```powershell
cd Pipeline\vivado_board
& 'C:\Xilinx\Vivado\2019.1\bin\vivado.bat' -mode batch -source .\run_board_synth_check.tcl
```

3. Optionally run synthesis for the PS/PL AXI-Lite wrapper:

```powershell
cd Pipeline\vivado_board
& 'C:\Xilinx\Vivado\2019.1\bin\vivado.bat' -mode batch -source .\run_axi_synth_check.tcl
```

The AXI-Lite wrapper simulation can be run from the LeNet testbench directory:

```powershell
cd Pipeline\testbench\testbench_LENET
powershell -NoProfile -ExecutionPolicy Bypass -File .\run_axi_lite_tb.ps1
```

The quick AXI register smoke test avoids the full LeNet runtime:

```powershell
cd Pipeline\testbench\testbench_LENET
powershell -NoProfile -ExecutionPolicy Bypass -File .\run_axi_lite_smoke.ps1
```

For the full LeNet run, Verilator is much faster:

```powershell
cd Pipeline\testbench\testbench_LENET
powershell -NoProfile -ExecutionPolicy Bypass -File .\run_axi_lite_verilator.ps1
```

4. Build the PS/PL block design wrapper for synthesis/HDF checking:

```powershell
cd Pipeline\vivado_board
& 'C:\Xilinx\Vivado\2019.1\bin\vivado.bat' -mode batch -source .\run_ps_bd_check.tcl
```

The generated AXI base address is `0x43C00000`.

The matching SDK 2019.1 baremetal polling example is in:

```text
Pipeline/sdk_app/rv32i_lenet_axilite/main.c
```

To run PS/PL implementation and generate a bitstream:

```powershell
cd Pipeline\vivado_board
& 'C:\Xilinx\Vivado\2019.1\bin\vivado.bat' -mode batch -source .\run_ps_bd_impl.tcl
```

Outputs:

```text
Pipeline/vivado_board/build_ps_bd_impl/rv32i_lenet_ps_bd.bit
Pipeline/vivado_board/build_ps_bd_impl/rv32i_lenet_ps_bd.hdf
```

For the multi-sample input-load variant:

```powershell
cd Pipeline\vivado_board
& 'C:\Xilinx\Vivado\2019.1\bin\vivado.bat' -mode batch -source .\run_ps_loadable_bd_impl.tcl
```

Sample-load AXI offsets from base `0x43C00000`:

```text
0x0000 control/status registers
0x1000 32x32 INT8 input image window, 1024 bytes
0x2000 expected label word
```

5. Copy `board_template.xdc` to `board.xdc`.
6. Fill in the real clock period, clock pin, reset pin, and LED package pins for
   your board.
7. Run the full standalone bitstream build:

```powershell
cd Pipeline\vivado_board
& 'C:\Xilinx\Vivado\2019.1\bin\vivado.bat' -mode batch -source .\run_board_impl.tcl
```
