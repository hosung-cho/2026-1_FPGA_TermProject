# RV32I LeNet AXI-Lite SDK App

This is a minimal Xilinx SDK 2019.1 baremetal application for the PS/PL block
design generated by:

```powershell
cd Pipeline\vivado_board
& 'C:\Xilinx\Vivado\2019.1\bin\vivado.bat' -mode batch -source .\run_ps_bd_check.tcl
```

The generated hardware handoff is:

```text
Pipeline/vivado_board/build_ps_bd_check/rv32i_lenet_ps_bd.hdf
```

Vivado 2019.1 exports an `.hdf` for SDK. Newer Vitis flows usually use `.xsa`.

## Address Map

The generated block design maps the RV32I AXI-Lite slave at:

```text
0x43C00000
```

Registers:

```text
0x00 control: bit0 run_enable, bit1 write-1 clear status
0x04 status : bit0 done, bit1 pass, bit2 fail
0x08 predicted digit
0x0C expected digit
0x10 heartbeat
0x14 debug_pc
0x18 debug_inst
0x1C debug_data_addr
0x20 debug_read_data
0x24 debug_write_data
0x28 debug_data_we
```

## SDK 2019.1 Use

1. Open Xilinx SDK 2019.1.
2. Import or create the hardware platform from
   `Pipeline/vivado_board/build_ps_bd_check/rv32i_lenet_ps_bd.hdf`.
3. Create a new standalone BSP and an Empty Application for `ps7_cortexa9_0`.
4. Copy `main.c` from this directory into the application's `src/main.c`.
5. Build and run on hardware.

Expected UART output for the bundled digit-7 firmware/data image:

```text
RV32I LeNet AXI-Lite test start
base=0x43C00000
...
done status=0x00000003 predicted=7 expected=7
PASS
```

If it times out, check that the bitstream is programmed, the PS FCLK is enabled,
and the AXI base address still matches the address editor.
