**Zynq7020 Bring-up 계획**  
PS+Vitis로 BRAM에 imem/dmem을 로드하고, `src/rtl` 기반 코어를 Zynq7020에 올려 검증까지 완료하는 흐름입니다.

1. Vivado/Vitis 버전을 확정합니다(2019.1 또는 2024.2로 통일).
2. Full E2E 기준으로 imem.hex/dmem.hex를 생성하고 Vitis용 C 배열로 변환합니다.
3. Zynq7020용 새 Vivado 프로젝트를 만들고 RTL을 추가한 뒤, BD에 Zynq PS + AXI BRAM Controller x2 + BRAM x2 + Clock Wizard + AXI GPIO(reset)를 구성합니다.
4. BRAM Port A는 PS AXI에, Port B는 `RV32I_System`의 imem/dmem 포트에 연결합니다. imem은 4096 words(16KB), dmem은 16384 words(64KB)로 설정하고 base address를 0xA0000000(imem), 0xA0010000(dmem)으로 매핑합니다.
5. 커스텀 보드 XDC를 적용하고 Clock Wizard 출력과 `CLOCK_50` 매핑을 정리합니다. reset은 AXI GPIO로 제어하고, `RV32I_System` 내부에서 reset이 반전되는 점을 고려합니다.
6. 합성/구현 후 bitstream 생성 및 XSA export를 진행합니다.
7. Vitis 플랫폼을 만들고 Hello World 템플릿을 바탕으로 호스트 앱을 작성합니다. BRAM에 inst/data를 로드 → D-cache flush → reset 토글 → status/결과 폴링 순서로 실행합니다.
8. status가 0x12345678이고 predicted label이 7인지 확인합니다. 실패 시 dmem 일부를 읽어 데이터 로딩 무결성을 점검합니다.

**검증**
1. Vivado Address Editor에서 imem/dmem base address와 BRAM 크기를 확인합니다.
2. Vitis에서 몇 개 BRAM 워드를 readback하여 로딩이 정상인지 확인합니다.
3. status=0x12345678, predicted label=7을 확인합니다.
