## 작업 내용

### 1. CNN Custom Function Unit (cnn_cfu.v) 구현

**파일 위치**: `src/rtl/cnn_cfu.v`

5개의 CNN 전용 연산을 순수 조합 로직으로 구현:

| 연산 | cfu_op | 기능 | 핵심 로직 |
|------|--------|------|-----------|
| MAC4 | `3'b000` | 4-way INT8 dot product | 4×(8b×8b→16b) + 4-input adder |
| RELU | `3'b001` | max(x, 0) | MSB 기반 MUX |
| MAXPOOL2 | `3'b010` | 4-input UINT8 max | 3-level comparator tree |
| CLRACC | `3'b011` | 상수 0 출력 | 직접 연결 |
| RESCALE | `3'b100` | (x+round)>>shift, clamp[0,255] | barrel shifter + clamp |

### 3. 발견한 버그 및 수정

#### RESCALE signed shift 버그

**증상**: TC24(음수 clamp), TC31(INT32_MIN clamp)이 FAIL
- 기대값: `0x00000000` (음수 → 0 clamp)
- 실제값: `0x000000FF` (255)

**원인**: `rounded`와 `shifted`가 `wire [31:0]` (unsigned)로 선언되어 있어서
`>>>` (arithmetic right shift)가 logical right shift로 동작. 음수 값이 shift 후 양수 큰 값이 되어
`>255` 조건에 걸려 255가 출력됨.

**수정**:
```verilog
// Before (버그)
wire [31:0] rounded;
wire [31:0] shifted;
assign rounded = rs1_data + rounding;
assign shifted = rounded >>> shift_amt;

// After (수정)
wire signed [31:0] rounded;
wire signed [31:0] shifted;
assign rounded = $signed(rs1_data) + $signed(rounding);
assign shifted = rounded >>> shift_amt;  // signed wire → arithmetic shift 정상 동작
```

추가로 비교 연산도 signed 비교로 수정:
```verilog
(shifted > 32'sd255)  // signed comparison
```
